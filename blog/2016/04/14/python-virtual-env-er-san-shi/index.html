Python Virtual Env 二三事

Apr 14th, 2016 3:18 pm

对python 开发者来讲，搭建virtual environment 是必然的实践。因为python库一是数量比较多，二是版本更新比较快。那么，搭建一个本地隔离的开发库非常有利于库的管理（防止和系统库混淆），也有利于多项目之间的相互隔离（一个项目的库不会污染全局库）。同时，这种隔离的库环境比较容易保证开发，测试，和生产环境使用同样的环境。基于以上的需要，python 的virtual environment 就应用而生了。

安装

python 2.x:
pip install virtualenv
python 3.x: 安装好python后自带了模块 venv 和 工具pyvenv，停供类似virtualenv 的使用方法

使用

具体使用方法可以查看文档

virtualenvwrapper

如果你有多个项目需要同时开发，那么需要一个有效的工具去在这些virtual env 之间进行切换，这是好virtualenvwrapper 便产生了。这款工具可以方便的在这些项目之间来回切换，自动加载相关开发设置。

pyenv

值得注意的是，pyvenv 是仅仅支持一个python 版本且限定于 3.x，那么该如何指定python 版本呢？ pyenv 你值得拥有，看名字就能猜到这个工具大概是效仿 renv 的。

venv-update

对于更多的virtual environment 的需求： 超大型项目需要维护一系列的 requirements，那么仅仅依靠virtual environment 很难满足类似requirement 库的维护：例如删除多余的库，校验库的完整性，自动升级库版本到指定的版本。。。这个时候你可以参考 Yelp 公司的一个开源项目 venv-update，该项目提供了对依赖库的管理工具 venv-update 以及依赖库的快速安装工具 pip-faster 可以助你一臂之力。

参考文档：

https://realpython.com/blog/python/python-virtual-environments-a-primer/
http://venv-update.readthedocs.org/en/stable/
